Utility classes CrudFaces provides several classes which provide useful general-purpose static util helper functionality.

In accordance with the “don’t reinvent the wheel” motto, CrudFaces promotes re-usage of existing functionality as e.g. provided by the vast amount of common “util” helper functions in OmniFaces’ helper classes. In fact, CrudFaces makes intense use of these helper functions to build its own functionality.

Whenever a requirement is found which is not covered by Java core, OmniFaces or PrimeFaces utils, but which seems to be commonly useful, CrudFaces will provide its own implementation as a helper util function.

So far, these util functions are covered by these three classes. In general, a util class name is based on its OmniFaces util class counterpart + Helper suffix.

  • ComponentsHelper: common JSF components util methods
  • FacesHelper: common JSF util methods
  • StringsHelper: simple String util methods for which including a full-blown String util library such as Apache common lang seems overkill.
