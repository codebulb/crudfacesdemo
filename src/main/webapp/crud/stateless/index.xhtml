Stateless view
    • A stateless combined master / detail view with support for AJAX actions.

    A CrudController implementation can be used to implement a stateless XHTML Facelets view.

    Here, the detail view of the currently “selected” entity is part of the master view (e.g. in a dialog box). Any CRUD action on an entity immediately updates the persistence storage.

    Usage

    Use these CrudController methods to trigger actions:

    • saveSelectedEntity(): Saves the selected entity
    • saveNewEntity(): Inserts the newEntity
    • saveSelectedOrNewEntity(): Saves the selected entity; or inserts the newEntity if none is selected
    • deleteEntity(id): Deletes the entity with id
    • deleteSelectedEntity() / deleteSelectedEntities(): Deletes the selected entity/ies

    The demo page contains info-boxes which explain the most important parts of building the XHTML Facelets view.

    Some of the basics about using CrudController as a backing bean are explained on the RESTful demo page.

    Of course, you can also view the entire demo page source code.

    Note: This showcases’ live demo uses a @SessionScoped CrudServiceMocked implementation instead of a true persistent service so that entities are kept in a HashMap in the user's session rather than in a database. Hence, you can invalidate your session in order to reset the entity collection.
