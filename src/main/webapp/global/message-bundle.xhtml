message-bundles
  • global overrides
    •  CrudFaces overrides several bean validation default messages with more reasonable and especially more end-user-friendly content.

    Some of the ValidationMessages’s default messages are either very technical and end-user-unfriendly by default, they lack a common sentence structure (e.g. some ending with a dot, others don’t), or have other design flaws.

    Most prominently, the javax.faces.validator.BeanValidator.MESSAGE String is overridden by CrudFaces such as to unify Apache MyFaces and Mojarra default implementation i.e. to always include the component’s label in a validation error message.

    Known restrictions

    • Providing an explicit <message-bundle> in web.xml overrides all default messages. As a partial workaround, Bean Validation message interpolation has been tweaked that it looks up validation message keys in <resource-bundle> as well. Hence, use your <resource-bundle> to define additional Bean Validation constraint annotation messages.
    • Still only a very limited amount of out-of-the-box message key / value pairs provided.
    • No I18N of those message keys out-of-the-box yet.

    These restrictions are most likely to be addressed and resolved in future CrudFaces versions.
