    •  CrudFaces initializes a web application with more reasonable, practice-proven default settings.

    CrudFaces comes out-of-the-box with a few JSF application configurations which derive from vanilla JSF configuration, but are considered best-practices and more useful for most JSF projects than are the actual JSF defaults.

    The following default <context-param> elements are added to the web.xml configuration:

    • javax.faces.FACELETS_SKIP_COMMENTS = true to disable the potentially harmful rendering of XHTML comments
    • javax.faces.INTERPRET_EMPTY_STRING_SUBMITTED_VALUES_AS_NULL = true to allow != null (@NotNull) checks rather than is empty (@NotEmpty) checks for input Strings.
    • primefaces.THEME = none as a preconfiguration step to use StyleClassChangeRenderKit.

    Of course, you can override these values again in your own web.xml.
