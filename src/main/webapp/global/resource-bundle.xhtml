    •  CrudFaces comes with several useful general-purpose I18N key / value pairs out of the box.

    In a typical JSF project, precious development time is wasted by defining commonly used message key / value pairs for every new project, e.g. “save”, “delete”, “cancel”, and so on. CrudFaces addresses this issue by delivering some commonly used messages out-of-the-box.

    Usage

    In order to use them alongside user-created keys, use them in the EL with the CrudFaces \#{I18N} bean rather than a reference to your resource-bundle var. By using the \#{I18N} bean, message keys are resolved in order their <resource-bundle>s are registered. Therefore, the default messages provided by CrudFaces are used as a fallback for message keys you don’t declare / overwrite explicitly.

    Known restrictions

    • Still only a very limited amount of out-of-the-box message key / value pairs provided.
    • No I18N of those message keys out-of-the-box yet.

    These restrictions are most likely to be addressed and resolved in future CrudFaces versions.
