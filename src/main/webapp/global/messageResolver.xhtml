OmniFaces message resolver
  • global overrides
    •  CrudFaces reconfigurates OmniFaces message resolution used by the Messages class helper methods to enable lookup by message key per default.

    When building FacesMessages with OmniFaces’ Messages class’s static helper methods, the message String provided is by default used as a hard-coded String.

    However, a real-life JSF application will most likely feature I18N-ready messages; hence, a lookup with a message key is more reasoanable than providing a hard-coded message as a String. CrudFaces uses OmniFaces’ own configuration facilities to override default message resolution with an actual message lookup.

    Here, logic from CrudFaces’ FacesHelper utility class is reused in order to implement a message lookup which correctly treats message-bundles from parent projects.
